Service caveat summary:

- Core safety behavior is concentrated in `smsSafety`, `messagingProvider`, Twilio webhooks, producer send readiness, and conversation live reply execution.
- Public-web functionality spans sourcing providers, async jobs, cleanup, and contactability. It is powerful but remains gated and disabled by default.
- Candidate graph/matching is real code with tests, but much of it is advanced/debug-facing until operator QA confirms the UX.
- The internal Saga API is a contract rehearsal. It must not be mistaken for production app integration.
