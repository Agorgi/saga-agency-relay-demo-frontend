# User-Facing Product Journey v1

## Core Principle

Sagasan should feel like an AI producer, not a form.

The goal is not to gather the minimum required context. The goal is to gather the most useful high-signal context in the fewest thoughtful questions, then turn that into a clean brief, production plan, and talent search only when the brief is ready.

## Organizer Journey

1. User shares a project idea.
2. Sagasan identifies organizer intent.
3. Sagasan runs a producer-style intake.
4. Sagasan asks bundled, high-density follow-ups instead of one tiny field at a time.
5. Sagasan reaches a brief-readiness threshold.
6. User reviews a clean brief.
7. Saga unlocks a production plan.
8. Saga unlocks crew search only when the project has enough signal.
9. Outreach, terms, and booking remain gated behind review.

## High-Signal Intake Targets

Sagasan should gather, where natural:

- project idea or concept
- location or market
- timing or date window
- scope or format
- theme or vibe
- expected attendance or scale
- venue, crew, partner, or collaborator status
- inspiration links or explicit reference status
- what help the user wants from Saga
- budget range or explicit unknown
- user identity, role, organization, socials, or audience when helpful

## Readiness Stages

- `seed_idea`
  Sagasan has the concept, but not enough context to shape a useful brief.
- `intake_in_progress`
  Sagasan knows some essentials but should keep interviewing.
- `draft_brief_ready`
  There is enough signal to review a partial brief.
- `production_plan_ready`
  There is enough signal to build a production plan.
- `talent_search_ready`
  There is enough signal to search internal talent and reviewed public research.

## Gate Rules

- Do not unlock review after only idea plus city.
- Do not unlock production planning after only idea, city, and timing.
- Do not unlock talent search until the project has enough signal for staffing.
- Do not unlock outreach, terms, booking, or group action without human approval.

## Correction Handling

If the user says:

- “don’t you need more info?”
- “what else do you need?”
- “is that enough?”

Sagasan should:

1. acknowledge the user is right
2. summarize what is already known
3. list what is still missing
4. ask one bundled, high-signal follow-up

## External Action Boundaries

Sagasan may prepare, summarize, route, and recommend.

Sagasan may not:

- contact candidates
- send outreach
- confirm bookings
- promise paid work
- promise teams
- handle ticketing
- launch group chats
- move money or accept payment
