  `GroupChat`, and/or `ProductionConversation`.

## 5. Internal API Contract

Auth model:

- All `/api/internal/saga/*` routes require `X-Saga-Internal-Key`.
- Missing or wrong keys return `401`.
- Current auth is shared-key MVP-grade and must be reviewed before production
  integration.

Routes:

- `POST /api/internal/saga/users/upsert`
- `POST /api/internal/saga/events/import`
- `POST /api/internal/saga/projects/:projectId/role-openings`
- `GET /api/internal/saga/projects/:projectId/recommendations`
- `GET /api/internal/saga/opportunities`
- `POST /api/internal/saga/opportunities/:opportunityId/interest`
- `POST /api/internal/saga/relationships/import`
- `POST /api/internal/saga/interest-checks`
- `POST /api/internal/saga/interest-checks/:id/interest`
- `POST /api/internal/saga/interest-checks/:id/convert`

Safe response rules:

- Do not expose `ADMIN_PASSWORD`, `INTERNAL_API_KEY`, provider tokens, database
  URLs, or raw secrets.
- Do not expose private contact notes, creator `internalNotes`,
  `availabilityNotes`, `rateNotes`, or admin notes to app-facing consumers.
- Return safe public fields, IDs, status, scores, score breakdowns, matching
  reasons, and risks.
- Do not expose raw production Saga ticketing, RSVP, QR, sales, or payment data.

Status:

- Internal API is fake/staging-only.
- It is not connected to the real Saga mobile app, desktop app, production
  database, ticketing, RSVPs, QR codes, event publishing, or payments.

## 6. Known Risks

Conversation Engine v0.1 is consolidated in
`docs/conversation-engine-v0.1.md`. It is active for mock/admin simulations,
but live Twilio behavior remains shadow-only and `SMS_SENDS_DISABLED=true`
continues to block real outbound sends. Do not treat the conversation engine as
approval for public SMS, autonomous outreach, or production app integration.

Producer Agent v0.1 is documented in `docs/producer-agent-v0.1.md`. It creates
draft-only project understanding, role maps, sourcing plans, internal candidate
recommendations, and shortlist drafts for admin review. It does not scrape the
web, contact candidates, send organizer shortlists, create group chats, or
connect to the production Saga app.

Producer Agent v0.2 is documented in `docs/producer-agent-v0.2.md`. It adds
candidate recommendation review states and durable shortlist packets for admin
approval. Approval means internal readiness only: it does not send SMS, create
outreach, contact organizers or candidates, or create group chats.

Producer Agent v0.3 is documented in `docs/producer-agent-v0.3.md`. It adds
review-only organizer shortlist and candidate outreach drafts. Draft approval
means copy readiness only: it does not send SMS, create outreach, contact
organizers or candidates, or create group chats.

- Shared-key internal API auth is MVP-grade.
- Admin auth is MVP-grade password/cookie auth.
- Production migrations need professional review before touching real user data.
- No live Twilio yet.
- No real Saga app integration yet.
- No public scraping.
- No autonomous outreach.
- Producer Agent recommendations, shortlist packets, and outbound drafts need
  professional review before any design-partner operating use.
- AI evals currently verify deterministic fallback behavior, not live model
  drift or provider latency.

## 8. Open Questions For Engineers

- Should canonical `Project` become the only source of truth after intake, with
  `ProjectBrief` treated strictly as transcript/brief metadata?
- What production auth should replace shared-key `X-Saga-Internal-Key`?
- Which real Saga user, event, community, friendship, ticketing, and RSVP fields
  may this service read, and which must remain inaccessible?
- What data retention and deletion policy applies to SMS/mock messages,
  outreach, creator profiles, and audit logs?
- Should phone/email ever be returned to the Saga app, or only displayed inside
  protected admin views?
- What approval workflow is required before a recommendation becomes an actual
  team member in the main Saga product?
- What thresholds should govern matching visibility: friends, mutuals,
  community, local, extended, public?
- How should opt-out and consent state synchronize if the existing Saga app adds
  app-native chat later?
- What is the operational owner for Twilio/A2P/10DLC compliance, privacy policy,
  opt-in language, and carrier approval?
- What observability, incident response, and audit export requirements are
  needed before production data is connected?
