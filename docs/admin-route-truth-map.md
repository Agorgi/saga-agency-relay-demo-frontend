- **Network Projects** is the canonical `Project` model page. It is useful but confusing beside Project Briefs. Future pass should fold it into Projects as a tab called "Canonical Projects" or merge detail pages.
- **Role Openings + Opportunities** represent two layers of staffing. They should become one operator-facing "Staffing Needs" page later.
- **Recommendations + Matching + Candidate Graph** should become one Smart Matching workbench with Candidate Graph under Advanced/debug.
- **Public Web Research + Research Cleanup** should become a tabbed Public Talent Research page once the review workflow stabilizes.
- **Needs Attention** is the operator queue for approvals/failures; it has no send controls.
